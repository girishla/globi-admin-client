Welcome to the SIL Workflow generation wizard. This wizard will allow you to select an existing metadata entity definition and generate Fact and Dimension workflows.
  • The Generator creates a Dimension SIL workflow with potentially 4 mappings.
    1. Mapping for Primary dimension table
    2. Mapping for Mini Dimension if metada contains attributes marked as "Mini"
    3. Mapping for Dimension extension if metadata contains attributes marked as "Legacy"
    4. Mapping to build Persistent, Named lookup cache for use by fact mappings
  • Resolves SCD logic & change capture audit attributes
  • Populates an unspecified row for each Dim table
  • Defaults NULL attributes to their corresponding Unspecified equivlents
  • Mapping compares MD5 Hash of content and updates only if Hash has changed
  • The Generator Creates a Fact SIL workflow with a single mapping
  • Each fact table is associated with a 1:1 Dim table. Please Ensure that the corresponding 1:1 Dimension Workflow is generated before the Fact
  • Fact tables can contain only Measure & Measure Attribute columns. They contain no system columns except ROW_WID
  • Mapping Resolves Wids for all Foreign Key columns including dates
  • Mapping Handles Currency Conversion into BU and Global Currencies
  • Expects that the metadata contains at least one Foreign Key and one Measure column
  • Mapping compares MD5 Hash of content and updates only if Hash has changed
  • Mapping sets ROW_WID to same as corresponding 1:1 dimension table
  • In addition to mapping Measures, also maps columns marked as "Measure Attributes"
